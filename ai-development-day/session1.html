🎯 CIFR Framework

📋 La estructura

  • Contexto: Dónde estás
  • Intención: Qué quieres lograr
  • Formato: Cómo lo quieres
  • Revisión: Qué validar

✨ Por qué funciona

Le das a la IA toda la información necesaria en un orden lógico que facilita generar código exactamente como lo necesitas.

Resultado: Menos iteraciones, más acierto en el primer intento.

💻 CIFR - Ejemplo real

🎯 Prompt estructurado con CIFR:

📍 CONTEXTO:
Estoy trabajando en un sistema de notificaciones para usuarios. Tengo un array de usuarios con emails y preferencias.

🎯 INTENCIÓN:
Necesito una función que filtre usuarios que tienen notificaciones activadas y envíe un email a cada uno usando nuestro servicio de email.

📐 FORMATO:
- Función async en TypeScript
- Con tipos bien definidos
- Manejo de errores con try/catch
- Logging de cada acción

✅ REVISIÓN:
Verifica que se manejen casos donde el email falla sin romper el loop para los demás usuarios.

✨ Resultado del prompt CIFR

// Tipos bien definidos
interface User {
  id: string;
  email: string;
  notificationsEnabled: boolean;
}

async function sendNotifications(users: User[]) {
  // Filtrar usuarios con notificaciones activadas
  const eligibleUsers = users.filter(
    u => u.notificationsEnabled
  );
  
  console.log(`Enviando a ${eligibleUsers.length} usuarios`);
  
  for (const user of eligibleUsers) {
    try {
      await emailService.send(user.email);
      console.log(`✅ Enviado a ${user.email}`);
    } catch (error) {
      console.error(`❌ Error con ${user.email}:`, error);
      // Continúa con el siguiente
    }
  }
}
🎯 Perfecto en el primer intento: Tipos, filtrado, manejo de errores, logging... todo lo que pedimos.

🐻 BEAR Framework

📋 La estructura

  • Background: Contexto técnico
  • Example: Un ejemplo claro
  • Action: Qué hacer
  • Result: Output esperado

🎯 Cuándo usarlo

Ideal cuando necesitas que la IA aprenda de un patrón o cuando quieres replicar un estilo específico de código.

Power move: Muy útil para transformaciones de datos o migraciones.

🐻 BEAR - Ejemplo real

🎯 Prompt estructurado con BEAR:

🎓 BACKGROUND:
Estamos migrando de un sistema de configuración JSON antiguo a uno nuevo basado en TypeScript. Tenemos 50+ archivos de configuración que migrar.

📝 EXAMPLE:
Formato antiguo:
{ "feature": "notifications", "enabled": true, "config": { "email": true } }

⚡ ACTION:
Crea una función que transforme este formato al nuevo esquema TypeScript usando clases y decorators.

🎯 RESULT:
Una clase FeatureConfig con tipo seguro y validaciones automáticas.

✨ Resultado del prompt BEAR

// Transformación automática JSON → TypeScript
class FeatureConfig {
  @IsString()
  feature: string;
  
  @IsBoolean()
  enabled: boolean;
  
  @ValidateNested()
  config: {
    email: boolean;
  };
  
  constructor(json: any) {
    this.feature = json.feature;
    this.enabled = json.enabled;
    this.config = json.config;
    this.validate();
  }
  
  private validate() {
    const errors = validateSync(this);
    if (errors.length > 0) {
      throw new Error(`Invalid config: ${errors}`);
    }
  }
}
🐻 Con un ejemplo claro: La IA entendió perfectamente el patrón y generó la estructura completa con validaciones.

🤖 Agent de Cursor

🤖

¿Qué es? Un asistente autónomo que puede leer archivos, hacer cambios, ejecutar comandos y razonar sobre tu código sin supervisión constante.

🎯 Capacidades del Agent

  • Autonomía: Toma decisiones sobre qué archivos leer/modificar
  • Contexto completo: Entiende la estructura del proyecto
  • Iteración: Puede probar, fallar y corregir automáticamente
  • Multi-archivo: Hace cambios coordinados en múltiples archivos
💡 Caso de uso real: "Refactoriza este componente para usar hooks en vez de clases, actualiza todos los archivos que lo importan y ajusta los tests"

🌍 Background Agent (Web)

🌐

El superpoder: El Agent puede buscar en internet en tiempo real para traer información actualizada, documentación de librerías, y soluciones a problemas específicos.

🚀 Qué puedes hacer

  • Documentación al instante: "Cómo usar la nueva API de Next.js 15"
  • Debugging: Busca soluciones a errores específicos
  • Best practices: Consulta las mejores prácticas actuales
  • Librerías nuevas: Aprende sobre paquetes que no conoce
🔥 Ejemplo: "Busca cómo implementar autenticación con PassKeys en 2025 y aplícalo a nuestro proyecto"

📊 Plan Mode

📋

Para tareas complejas: Antes de ejecutar, el Agent crea un plan detallado de todos los pasos que va a realizar. Tú lo revisas y apruebas.

✅ Ventajas del Plan Mode

  • Visibilidad: Ves exactamente qué va a hacer antes de hacerlo
  • Control: Puedes ajustar el plan antes de ejecutar
  • Aprendizaje: Entiendes la estrategia del Agent
  • Seguridad: Evitas cambios no deseados en tareas críticas
💼 Caso de uso: Migraciones grandes, refactorizaciones críticas, o cuando trabajas en producción

👥 Multiple-agent

🎭

Paralelización inteligente: Puedes ejecutar múltiples Agents a la vez, cada uno trabajando en tareas diferentes de forma simultánea.

⚡ Productividad x10

  • Agent 1: Refactoriza el backend mientras...
  • Agent 2: Actualiza los tests y...
  • Agent 3: Genera documentación
  • Resultado: 3 tareas terminadas en el tiempo de 1
🎯 Pro tip: Usa esto cuando tienes tareas independientes. Ej: "Agent 1: implementa feature X" y "Agent 2: arregla bugs en la lista"

🛡️ Allowlist

🔐

Control de seguridad: Define exactamente qué archivos, carpetas o comandos puede tocar el Agent. Lo demás está prohibido.

🎯 Configuración inteligente

  • Por carpetas: "Solo puedes modificar /src y /tests"
  • Por tipo: "No toques archivos .env o .config"
  • Por comandos: "No ejecutes npm install sin permiso"
  • Whitelist: Archivos críticos quedan protegidos
💼 Escenario real: Trabajas en un monorepo grande pero solo quieres que el Agent modifique tu microservicio, no todo el repo

📏 Cursor Rules

Tu manual de instrucciones personalizado: Configura una vez, aplica siempre. El Agent seguirá tus reglas en cada conversación.

Dos tipos de Cursor Rules:

  • 📚 Guidelines: Estándares de código, patrones, convenciones
  • 🎯 Prompts: Instrucciones específicas de cómo trabajar

📚 Cursor Rules: Guidelines

// .cursorrules - Guidelines para el proyecto

# Estándares de código
- Usa TypeScript estricto (strict mode)
- Nombres de variables en camelCase
- Componentes React en PascalCase
- Archivos de tests con .test.ts

# Arquitectura
- Sigue clean architecture
- Separa lógica de negocio de UI
- Usa dependency injection

# Imports
- Imports absolutos con @/ alias
- Agrupa: externos → internos → tipos

# Testing
- Test unitario para cada función pública
- Coverage mínimo 80%
- Usa testing-library para React
💡 El Agent generará código siguiendo SIEMPRE estas reglas sin que las repitas

🎯 Cursor Rules: Prompts

// .cursorrules - Instrucciones de comportamiento

# Commits

📂 MCPs - Model Context Protocol

El verdadero game changer: MCPs permiten que Cursor se conecte con servicios externos en tiempo real. Es como darle superpoderes al Agent.

¿Qué son los MCPs?

Protocolos estandarizados que conectan Cursor con APIs externas, bases de datos, browsers, herramientas... básicamente cualquier cosa.

Vamos a ver:

  • 🌐 Browser MCP (integrado en Cursor)
  • 🔌 Conectar MCPs externos
  • 🛠️ Crear MCPs custom para developers

🌐 MCP: Browser

🌐

Ya integrado en Cursor: El Agent puede controlar un browser headless, navegar páginas web, llenar formularios, scrappear datos...

🎯 Casos de uso reales

  • Testing E2E automático: "Navega a login, rellena el form, verifica el resultado"
  • Web scraping: "Extrae los precios de esta página y guárdalos en CSV"
  • Investigación: "Busca en la documentación de X cómo implementar Y"
  • Screenshots: "Captura cómo se ve la app en mobile y desktop"
🔥 Ejemplo potente: "Abre nuestra app en producción, haz el user flow completo y dime si hay errores en consola"

🔧 MCP Custom: Código base

// mcp-n8n-server.ts - Ejemplo simplificado
import { MCPServer } from "@modelcontextprotocol/sdk";

const server = new MCPServer({
  name: "n8n-integration",
  version: "1.0.0"
});

// Definir herramienta: ejecutar workflow
server.addTool({
  name: "execute_n8n_workflow",
  description: "Ejecuta un workflow de n8n",
  parameters: {
    workflowId: "string",
    data: "object"
  },
  handler: async ({ workflowId, data }) => {
    const result = await fetch(
      `https://n8n.housfy.com/webhook/${workflowId}`,
      { method: "POST", body: JSON.stringify(data) }
    );
    return result.json();
  }
});

server.start();
🎯 Con ~30 líneas de código tienes un MCP funcional. Luego lo configuras en Cursor y... ¡magia!

📝 Codex (ChatGPT)

📚

No es de Cursor, pero es oro: Codex de ChatGPT es un sistema de memoria compartida donde guardas snippets, configuraciones, patrones... y los reutilizas en todos tus chats.

💎 Por qué es increíble

  • Memoria persistente: Ya no explicas lo mismo 100 veces
  • Snippets reutilizables: "Usa mi configuración de ESLint guardada"
  • Patrones de tu código: "Sigue el patrón de autenticación que tengo en Codex"
  • Cross-projects: Reutiliza conocimiento entre proyectos
💡 Pro tip: Guarda en Codex: configs, arquitectura de tus proyectos, decisiones técnicas, y tus reglas de negocio complejas

📚 Codex - Casos de uso

💼 Qué guardar en tu Codex personal:

🏗️ Arquitectura del proyecto:
"Nuestra app usa clean architecture con capas: domain, application, infrastructure. Domain no puede importar de las otras"

🎨 Convenciones de diseño:
"Todos los botones primarios usan el color #667eea, border radius de 8px, shadow-sm en hover"

📋 Reglas de negocio:
"Los usuarios premium tienen acceso a X, Y, Z. La lógica de validación está en UserPermissionsService"

🔧 Setup del proyecto:
"Para local: npm install, copiar .env.example, ejecutar docker compose, seed de base de datos"
